No começo (fevereiro) de 2011 o [Hanselman](http://www.hanselman.com/blog/) publicou um [post](https://www.hanselman.com/blog/new-interview-questions-for-senior-software-engineers) contendo diversas perguntas que ele considera úteis para uma entrevista com um engenheiro de software Senior.

Para quem tiver a intenção de utilizá-la para avaliar seus desenvolvedores Seniores é bem interessante, porém principalmente para quem pretende tornar-se um Senior, essa lista pode servir como guia, e além disso como parâmetro de autoavaliação.

A lista original (em Inglês) pode ser encontrada no próprio post dele. Abaixo segue a tradução (não literal) que fiz da lista de perguntas*:

- Qual foi a coisa substancial que você fez em sua carreira para melhorar como desenvolvedor?
- Would you call yourself a craftsman (craftsperson) and what does that word mean to you?
- Implemente uma \<estrutura básica de dados\> usando \<alguma linguagem\> no \<papel\|quadro\|notepad\>
- O que é SOLID?
- Por que o Princípio da Responsabilidade Única (Single Responsibility Principle) é importante?
- O que é Inversão de Controle (Inversion of Control)? Como isto se relaciona com injeção de dependência (dependency injection)?
- O que difere uma aplicação de 3 camadas de uma de 2 camadas?
- Por que as interfaces são importantes?
- O que é o padrão Repositório (Repository)? E o padrão Fábrica (Factory)? Por que os padrões são importantes?
- Quais são alguns exemplos de anti-patterns?
- Quem são os Gang of Four? Por que importar-se?
- Como os padrões MVP, MVC e o MVVM se relacionam? Quando eles são apropriados?
- Explique o conceito de Separation of Concerns e os seus pros e contras.
- Nomeie três atributos primários de projetos orientados a objeto. Descreva o que eles significam e por que eles são importantes.
- Descreva um padrão que NÃO seja o Factory pattern? Como e quando usá-lo?
- Você acaba de ser encarregado de um projeto de código legado com problemas de manutenibilidade. Que tipo de coisas você faria para melhorar o projeto até chegar a uma base estável?
- Mostre-me um portfolio de todas as aplicações que você já trabalhou e conte-me como você contribuiu para o projeto (design) das mesmas.
- Quais são algumas das alternativas para armazenar dados que sejam diferentes de bancos de dados relacionais? Por que você faria isso e quais são os pontos negativos?
- Explique o conceito de convenção sobre configuração e fale de um exemplo de convenção sobre configuração que você já viu em produção.
- Explique a diferença entre sistemas stateless e stateful e os impactos do estado (state) no paralelismo.
- Discuta a diferença entre Mocks e Stubs/Fakes e onde você pode usar eles (respostas não são tão importantes aqui, apenas a discussão que acontecer).
- Discuta o conceito de YAGNI e explique algo que você fez recentemente que aderiu a esta prática.
- Explique o que você entende por sandbox, por que você deve usar e identifique exemplos de sandboxes em produção.
- Concorrência. Qual a diferença os modelos de concorrência Locking e Lockless (Otimista e Pessimista).
- Que tipo de problemas você pode ter com modelo locking? E com um modelo lockless?
- What trade offs do you have for resource contention?
- Como um modelo baseado em tarefas se difere de um modelo em threads?
- Qual é a diferença entre assincronia e concorrência?
- Você continua escrevendo código? Você ama fazer isso?
- Você acaba de ser atribuído a um projeto em/com uma nova tecnologia, como você começaria?
- Como a adição de Orientação a Serviço altera os sistemas? Quando é apropriado utilizá-la?
- O que você faz para ficar a par das últimas tecnologias e ferramentas?
- Qual é a diferença entre lógica "set" e lógica "procedural". Quando e por que você usa cada uma delas?
- Quais sistemas de Source Control você já trabalhou?
- O que é Integração Contínua? Você têm usado e por que é importante?
- Descreva um ciclo de vida de um software que você gerenciou.
- Como você reage quando as pessoas criticam seu código ou seus documentos?
- Quais blogs ou podcasts você segue? Você publica em algum blog ou podcast?
- Fale-me sobre alguns de seus projetos de passatempo (hobby) que você escreveu em seu tempo livre.
- Qual é o último livro de programação que você leu?
- Descreva, com o máximo de detalhes que acha relevante, tão profundamente quanto puder, o que acontece quando eu digito "cnn.com" em um navegador e pressiono "Ir".
- Descreva a estrutura e o conteúdo de um documento de design, ou um conjunto de documentos de design, para uma aplicação web de várias camadas.
- O que há de tão incrível na \<tecnologia web atual legal\>?
- How can you stop your DBA from making off with a list of your users' passwords?
- O que você faz quando fica preso em um problema que não pode resolver?
- Se o seu banco de dados estiver sob muita tensão, quais são as primeiras coisas que você consideraria para acelerá-lo?
- O que é SQL injection?
- Qual é a diferença entre teste de unidade e teste de integração?
- Fale-me sobre umas 3 vezes que você falhou.
- O que é Refactoring? Você têm usado? Isso é importante?
- Nomeie três refatorações comuns.
- Você tem dois computadores e você deseja obter dados de um para o outro. Como você faria isso?
- Left to your own devices, what would you create?
- Dado Tempo, Custo, satisfação do Cliente, e Melhores Práticas. Como você os prioriza num projeto que você está trabalhando? Explique o porquê.
- Qual a diferença entre web server, web farm e web garden? Como sua aplicação web poderia mudar entre cada um?
- Qual o valor que compilações diárias, testes automatizados e revisões em pares adiciona para um projeto? Quais desvantagens existem nisso?
- Quais elementos de design OO são mais propensos a serem abusados? Como você pode mitigar eles?
- Quando você sabe que seu código está pronto para ir para produção?
- O que é YAGNI? Esta lista de perguntas é um exemplo?
- Descreva-me algum código ruim que você leu ou herdou recentemente.

\* Algumas perguntas achei melhor manter em Inglês, pois não encontrei uma tradução que mantivesse exatamente o mesmo sentido.
